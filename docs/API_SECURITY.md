# Безопасность API ключей

## Что было сделано (Декабрь 2025)
В целях безопасности все API ключи (TMDB, Kinopoisk) были удалены из исходного кода проекта. Это позволяет безопасно публиковать код на GitHub, не боясь утечки платных или личных ключей.

### Текущая структура
1. **Хранение ключей**: Ключи хранятся только в файле `.env` в корне проекта. Этот файл добавлен в `.gitignore` и не попадает в репозиторий.
2. **Клиент (Frontend)**: Использует переменные окружения `VITE_TMDB_API_KEY` и `VITE_KP_API_KEY`.
    - При сборке проекта (`npm run build`) Vite подставляет реальные значения из `.env` в код.
    - В исходниках (`App.jsx`) ключей нет, там только ссылки `import.meta.env...`.

## Инструкция на будущее

### Если вы переустанавливаете проект или меняете сервер
1. Создайте файл `.env` в корне проекта (можно скопировать из `.env.example`).
2. Заполните ключи:
   ```env
   VITE_TMDB_API_KEY=ваш_ключ
   VITE_KP_API_KEY=ваш_ключ
   ```
3. Перезапустите сервер (чтобы он увидел файл).

### Если вы вносите правки в Frontend (папка client)
Так как ключи "впекаются" в статику при сборке, если вы хотите обновить код клиента или поменяли ключи:
1. Убедитесь, что файл `.env` с ключами существует.
2. Запустите сборку:
   ```bash
   cd client
   npm run build
   ```
   Эта команда создаст новую версию сайта в папке `dist`, уже с вашими ключами.
